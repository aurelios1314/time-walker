## 2. 产品概述

### 2.1 产品背景与目标
《时行》面向普通用户，帮助他们在做每日计划和决策时，能方便地参考多维度的背景信息——当天的天气、所处节气、临近节日、个人运势参考以及当日待办——而不需要分散打开多个 App。

目标是让每个人都能轻松用上传统历法工具和 AI 分析，而不需要有任何命理或历法的专业背景。

### 2.2 整体布局架构
整体采用**左侧导航栏 + 中央大看板 + 右侧对话框**的三栏布局。

| 区域 | 角色 | 说明 |
|---|---|---|
| 左侧导航栏 | 导航入口 + 摘要展示 | 各功能以小卡片形式展示关键信息摘要，点击后在中央大看板展开完整内容 |
| 中央大看板 | 动态内容区 | 根据左侧当前选中的功能，展示对应的完整内容，默认展示日历 |
| 右侧对话框 | AI 对话助手 | 固定显示，不随左侧切换变化 |

**核心交互规则**：
*   左侧每张卡片同时具备两个功能：① 随时展示该功能的实时数据摘要；② 点击后切换中央大看板到该功能的完整内容。
*   同一时刻只有一个卡片处于选中状态，大看板对应只展示一个功能。
*   页面打开时默认展示日历。

### 2.3 技术说明
*   **架构模式**：纯前端单页应用，无需服务器，可本地运行。
*   **开发语言**：HTML5, CSS3, JavaScript (ES6+)。
*   **核心依赖库**：`lunar-javascript`（提供公历/农历互转、节气及传统节日数据）。
*   **数据存储**：所有用户数据均保存在浏览器本地存储（`localStorage`）中，不上传任何服务器。

## 3. 功能模块详细需求

### 3.1 左侧导航栏

左侧由多个功能卡片纵向排列，每张卡片：
- 始终展示该功能的核心数据摘要（用户无需点击即可快速扫读）
- 点击后切换中央大看板展示该功能的完整内容

#### 导航项清单

| 卡片名称 | 卡片上展示的摘要 | 点击后大看板展示的内容 |
|---|---|---|
| 日历 | 当前年月、今日农历日期 | 完整月历（默认） |
| 天气 | 天气图标、今日温度区间、当前城市 | 今日天气 + 24小时逐时预报 |
| 节气 | 当前节气或距下一个节气的天数 | 节气详情 + 全年节气列表 |
| 节日 | 最近的公历节日和农历节日倒计时 | 未来 180 天节日完整列表 |
| 我的信息 | 出生日期对应的生命周期标签摘要 | 完整个人信息档案 + 当日随笔 |
| 事项管理 | 今日待办数 / 今日总事项数 | 当日完整事项列表 + 新增入口 |
| 天时洞察 | AI 密钥状态（已配置 / 未配置） | AI 分析结果面板 |

---

### 3.2 中央大看板

根据左侧选中的功能切换显示，面板切换时有 0.3s 的淡入动效。

#### 3.2.1 日历（默认）

*   **月历网格**：
    *   以 7 列格网排列当月所有日期，每格同行显示公历日期（大字）和农历日期（小字），两者底部对齐。
    *   若当日有公历节日，在公历数字下方以灰色小字展示节日名称，允许换行，不做截断。
    *   若当日有农历节日或节气，在农历日期下方展示：节气用红色标注，农历节日用金色标注。
    *   非本月日期（上/下月溢出格）显示为半透明灰色。
*   **导航**：提供「上个月」「下个月」「回到今天」三个按钮。页面打开时默认选中今天（红色圆圈高亮）。
*   **日期点击**：点击任意日期格子选中该日，左侧所有卡片同步刷新为该日的数据。
*   **事项标记点**：每个日期格子下方用小圆点标记当天的事项，重要事项用红点，普通事项用橙点。

#### 3.2.2 天气

*   展示当前定位城市今日天气：天气图标、气温范围、天气描述。
*   以卡片流展示今日 24 小时逐时天气（时刻、温度、天气图标）。
*   定位策略：优先使用浏览器定位；若被拒绝或失败，自动改用 IP 定位（`ipapi.co`）；均失败则默认北京坐标。

#### 3.2.3 节气

*   展示当前选中日期的节气情况：
    *   若当日恰好是某个节气，高亮提示当日节气名称。
    *   展示下一个节气名称、距今天数和具体公历日期。
*   展示当年剩余所有节气的名称和日期列表。

#### 3.2.4 节日

*   从当前选中日期起，向后 180 天内，分两栏展示：
    *   **公历节日**：元旦、劳动节、国庆节等，逐条列出名称和倒计时天数。
    *   **农历节日**：春节、中秋节、端午节等，逐条列出名称和倒计时天数。

#### 3.2.5 我的信息

*   **个人档案展示**：展示用户保存的个人信息，包括：
    *   出生日期对应的传统历法信息（以用户能理解的方式呈现，避免堆砌专业术语）。
    *   用户自行勾选的性格、心理状态、健康状态标签。
*   **当日随笔**：展示当前选中日期的个人备注内容，支持直接在此编辑和保存。
*   **编辑入口**：「编辑我的信息」按钮，点击弹出编辑表单（性别、出生日期、出生时间、AI 密钥）。

#### 3.2.6 事项管理

*   **当日事项列表**：展示选中日期的所有事项卡片，每条包含标题、时间段、分类标签、是否重要标记、完成状态勾选框。
*   **新增事项**：「+ 新增」按钮，弹出表单（标题必填、时间段可选、分类可选、是否重要开关）。
*   **完成状态**：点击勾选框即时更新完成状态，并同步刷新日历面板上的事项标记点。

#### 3.2.7 天时洞察

*   **趋势曲线**：以折线图展示当日 24 小时的综合状态指数（0-100），支持「横向展开」和「圆形时钟」两种视图切换。
*   **洞察卡片**：在曲线的特定时间点放置 4 张分析卡片，点击可翻开查看具体分析内容。
*   **AI 生成**：
    *   若当日已有缓存结果，直接展示，无需重新请求。
    *   若无缓存，自动向 AI 接口（DeepSeek Chat）发起请求，将当日天气、节气、事项清单、个人状态备注作为背景信息，生成 4 张分析卡片的内容，并缓存到本地。

---

### 3.3 右侧对话框

固定显示在右侧，不随左侧导航切换而变化。

*   **对话记录**：以气泡列表展示历史对话。用户发言右对齐（红色气泡），AI 回复左对齐（米黄色气泡）。
*   **发送消息**：文本框输入，支持回车键或点击发送按钮提交，空内容不可发送。
*   **AI 回复**：用户发送后 1 秒内自动生成一条 AI 回复，并滚动到最新消息。
*   **清空对话**：右上角清空按钮，点击后清除所有历史记录，恢复初始欢迎语。

## 4. 数据存储说明

所有数据保存在浏览器本地，不上传服务器。

### 4.1 事项列表 (`shixing_tasks`)
```json
[
  {
    "date": "2026-05-20",
    "title": "整理产品需求文档",
    "time": "15:00",
    "tag": "工作",
    "isImportant": true,
    "completed": false
  }
]
```

### 4.2 每日随笔 (`shixing_daily_feelings`)
```json
{
  "2026-05-20": "今天状态不错，思路清晰。"
}
```

### 4.3 个人信息档案 (`shixing_profile`)
```json
{
  "gender": "男",
  "date": "1990-05-20",
  "time": "12:00",
  "bazi": "<div>...</div>",
  "baziText": "日柱：丙寅",
  "personality": "坚韧、敏感",
  "psych": "专注",
  "health": "精力足"
}
```

### 4.4 AI 密钥 (`shixing_api_key`)
```
"sk-********************************"
```
明文存储在本地，仅用于向 AI 接口鉴权，不会传输到其他地方。

### 4.5 AI 洞察缓存 (`shixing_tarot_2026-5-20`)
每日一条，包含 4 张分析卡片的标题和内容。
```json
[
  { "type": "yun", "title": "今日整体状态", "content": "..." },
  { "type": "mou", "title": "行动建议", "content": "..." },
  { "type": "xin", "title": "身心提示", "content": "..." },
  { "type": "zhi", "title": "环境参考", "content": "..." }
]
```

## 5. 设计规范

*   **配色**：以暖色调为主，背景米白（`#fffdf9`），主文字深棕（`#2f2522`），强调色深红（`#8c2230`），点缀金色（`#d4af37`）。
*   **动效**：大看板面板切换 0.3s 淡入；弹窗 0.4s 滑入；气泡和卡片翻转 0.3s 过渡。
*   **移动端**：视口宽度 ≤ 1024px 时，三栏布局折叠为单列垂直排列。
